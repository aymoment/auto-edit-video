## 环境准备

### 系统依赖

必须安装 **ffmpeg** 和 **ffprobe**，用于音频提取和视频渲染。

```bash
# macOS
brew install ffmpeg

# Ubuntu/Debian
sudo apt install ffmpeg

# Windows
# 下载 ffmpeg 并添加到 PATH 环境变量
```

### Node.js

需要 Node.js 18+ 和 pnpm 包管理器。

```bash
# 安装 pnpm（如已安装可跳过）
npm install -g pnpm@9
```

---

## 快速开始

```bash
# 1. 安装依赖
pnpm install

# 2. 构建共享类型包
pnpm -C packages/shared build

# 3. 一键启动前后端（后端 4000 端口 + 前端 5173 端口）
pnpm dev
```

也可以分别启动：

```bash
# 终端 1：启动后端
pnpm dev:api

# 终端 2：启动前端
pnpm dev:web
```

打开浏览器访问 **http://localhost:5173**。

## 完整工作流

### 1. 创建项目并上传视频

在首页点击**「创建新项目」**按钮，进入编辑器页面。

在编辑器左侧面板底部点击**「上传视频」**，选择本地的口播视频文件（支持常见视频格式：mp4、mov、avi 等）。

上传成功后，左侧面板会显示视频播放器。

### 2. 解析视频

上传视频后点击**「开始解析」**按钮。系统会：

1. 从视频中提取音频
2. 调用转录服务（Mock 或火山引擎）进行语音识别
3. 根据静音阈值切分片段
4. 根据相似度检测重复片段
5. 将视频分割为若干片段，标记为**正常 / 静音 / 重复**

解析完成后的片段会自动显示在右侧面板中。默认所有正常片段为选中状态，静音和重复片段为未选中。

### 3. 筛选片段

在右侧面板顶部有三个 Tab 切换：

- **全部** — 显示所有片段
- **已选** — 只显示已勾选的片段（将出现在最终输出中）
- **未选** — 只显示未勾选的片段

点击片段卡片右上角的复选框来 **选中 / 取消选中** 该片段。

中间预览区会实时展示已选片段拼接后的效果。

### 4. 编辑片段

对每个**正常类型**的语音片段，可以进行以下操作：

- **预览** — 在左侧原始视频播放器中播放该片段对应的时间段
- **编辑** — 修改该片段的字幕文本。编辑后会保存为 `editedText`，导出时优先使用编辑后的文本
- **拆分** — 将一个片段拆分为多个小片段（详见[拆分片段](#拆分片段)）

### 5. 调整裁剪比例

在左侧面板的「视频比例」区域，选择输出视频的裁剪比例：

| 选项     | 说明                   |
| -------- | ---------------------- |
| 原始尺寸 | 保持原视频比例         |
| 16:9     | 横屏，适合 B站/YouTube |
| 9:16     | 竖屏，适合 抖音/小红书 |
| 4:3      | 传统比例               |
| 3:4      | 竖屏比例               |
| 自由裁剪 | 暂未开放               |

选择比例后，中间预览区会实时更新显示效果。

### 6. 配置字幕样式

点击右侧面板底部的**「字幕样式」**按钮，打开字幕样式弹窗。

可配置的项目：

| 分类 | 参数                    | 说明                                      |
| ---- | ----------------------- | ----------------------------------------- |
| 字体 | 字体大小                | 20-72px，滑块调节                         |
| 字体 | 字体粗细                | Regular / Medium / Semibold / Bold        |
| 字体 | 字体系列                | 中文默认 / Noto Sans SC / Arial / Georgia |
| 动画 | 入场/出场               | 淡入淡出 / 上移 / 下移 / 缩放 / 无        |
| 文字 | 文字颜色                | 颜色选择器                                |
| 描边 | 描边颜色 / 粗细         | 0-8px                                     |
| 背景 | 背景颜色 / 透明度       | 0-1 可调                                  |
| 位置 | 底部位置                | 2%-20% 距底部距离                         |
| 阴影 | 阴影颜色 / 模糊 / Y偏移 | -20 ~ 20px                                |

修改后点击**「保存」**，预览区会立即生效。

### 7. 设置渲染参数

在左侧面板配置最终输出视频的参数：

**帧率**

- 30 FPS（默认，适合大多数场景）
- 60 FPS（更流畅，文件更大）

**清晰度**

- 720p — 快速预览，文件小
- 1080p — 全高清，推荐
- 2K — 超高清

> 清晰度仅影响最终渲染导出，不影响实时预览。

**渲染质量**

- 标准 — 码率 ×1.0
- 高质量 — 码率 ×1.3
- 超高 — 码率 ×1.6

系统会根据所选清晰度、帧率和质量自动计算预计码率和输出分辨率。

### 8. 渲染导出

确认片段选择和参数配置无误后，点击底部工具栏的**「渲染视频」**按钮。

渲染过程会显示进度条遮罩，完成后会提示输出路径，下载按钮变为可用。

点击**「下载视频」**即可将最终视频保存到本地。

渲染使用的视频编码为 H.264，容器格式为 MP4。

## 高级功能

### 拆分片段

将一个语音片段拆分为多个小片段。

操作步骤：

1. 在片段卡片上点击**「拆分」**按钮
2. 在弹出对话框中，在文本中合适的位置插入 `#` 作为分割标记
3. 右侧会实时显示拆分预览（每段文字编号列出）
4. 点击**「拆分」**确认

拆分逻辑：

- 优先根据原始转录的时间戳精确定位拆分点
- 如果文本已被编辑过，会提示使用近似时间拆分（按文字长度比例分配时长）

### 合并片段

将相邻片段合并为一个。

操作步骤：

1. 点击右侧面板底部的**「合并片段」**
2. 在列表中选择 2 个或更多**位置相邻**的片段
3. 点击**「合并」**确认

> 合并只能选择位置连续的片段，不支持跨片段选择。

### 移动片段

调整片段的排列顺序。

操作步骤：

1. 点击**「移动片段」**
2. 通过拖拽手柄调整片段顺序
3. 点击**「确定」**保存

支持按 Tab（全部/已选/未选）筛选后再拖拽。

### 批量替换文案

在多个片段的字幕文本中批量查找并替换文字。

操作步骤：

1. 点击**「批量替换」**
2. 输入**原始词**（要查找的内容）和**替换词**
3. 系统自动搜索所有匹配的片段，显示匹配列表（高亮匹配文字）
4. 勾选/取消勾选要替换的片段
5. 可选开启**「忽略大小写」**
6. 点击**「确认替换」**

### 导出字幕

将已选片段的字幕文本导出为文件。

支持三种格式：

- **SRT** — 通用字幕格式，兼容大多数视频播放器
- **VTT** — WebVTT 格式，适合网页播放
- **TXT** — 纯文本，仅包含字幕文案

操作步骤：

1. 点击**「导出字幕」**
2. 选择字幕格式
3. 点击**「下载」**

## 全局设置

点击任意页面右上角的齿轮图标进入设置。

### 解析参数

| 参数       | 范围       | 默认值 | 说明                                                 |
| ---------- | ---------- | ------ | ---------------------------------------------------- |
| 静音阈值   | 100-2000ms | 500ms  | 静音时长超过此值则切分为独立静音片段。值越小越敏感   |
| 重复相似度 | 50%-99%    | 默认值 | 两段文字相似度超过此值则标记为重复。值越高要求越严格 |

> 修改后需要**重新解析**才会生效。

### 转录服务

| 选项         | 说明                                                                                     |
| ------------ | ---------------------------------------------------------------------------------------- |
| **火山引擎** | 使用火山引擎语音识别服务，需要填写 API Key、Resource ID、Submit Endpoint、Query Endpoint |
| **Mock**     | 本地模拟转录，用于开发调试，无需联网                                                     |

### 上传方式

| 选项              | 说明                                       |
| ----------------- | ------------------------------------------ |
| **Uguu 临时上传** | 视频上传到 Uguu 临时存储服务               |
| **本地公网直链**  | 直接使用本地文件路径，需配置公网访问根地址 |

## API 接口

后端运行在 `http://localhost:4000`。

| 方法     | 路径                                     | 说明                                         |
| -------- | ---------------------------------------- | -------------------------------------------- |
| `POST`   | `/api/projects`                          | 上传视频并解析。FormData：`file`（视频文件） |
| `GET`    | `/api/projects`                          | 获取所有项目列表                             |
| `GET`    | `/api/projects/:id`                      | 获取单个项目详情                             |
| `PATCH`  | `/api/projects/:id/segments`             | 保存片段选择/文本修改                        |
| `PATCH`  | `/api/projects/:id/title`                | 修改项目名称                                 |
| `PATCH`  | `/api/projects/:id/crop`                 | 修改裁剪设置                                 |
| `PATCH`  | `/api/projects/:id/render-settings`      | 修改渲染参数                                 |
| `POST`   | `/api/projects/:id/reanalyze`            | 重新解析视频                                 |
| `POST`   | `/api/projects/:id/render`               | 触发 Remotion 渲染导出                       |
| `DELETE` | `/api/projects/:id`                      | 删除项目                                     |
| `GET`    | `/api/projects/:id/video`                | 获取原始视频文件                             |
| `GET`    | `/api/projects/:id/output`               | 获取渲染输出文件                             |
| `GET`    | `/api/projects/:id/subtitles?format=srt` | 导出字幕（srt/vtt/txt）                      |
| `GET`    | `/api/settings`                          | 获取全局设置                                 |
| `PUT`    | `/api/settings`                          | 更新全局设置                                 |

## 技术架构

```
apps/
  api/        NestJS 后端 — 视频处理、转录分析、渲染调度
  web/        React 前端 — Vite + shadcn/ui + Tailwind CSS
  render/     Remotion Composition — 视频预览与渲染模板
packages/
  shared/     共享 TypeScript 类型定义
```

- **前端**：React 18 + Vite 6 + shadcn/ui（Radix UI + Tailwind CSS）+ Remotion Player
- **后端**：NestJS + Express + Multer（文件上传）+ Remotion Renderer
- **渲染引擎**：Remotion（基于 FFmpeg 的视频合成）
- **数据存储**：本地文件系统（`data/` 目录），项目元数据为 JSON 文件
- **包管理**：pnpm workspace monorepo

### 环境变量

```bash
# mock | volcengine
TRANSCRIBE_PROVIDER=mock

# 当 TRANSCRIBE_PROVIDER=volcengine 时需要
VOLCENGINE_ASR_ENDPOINT=https://example.com/asr
VOLCENGINE_ASR_TOKEN=your_token
```

### 目录结构

```
data/
  projects/        项目 JSON 元数据
  uploads/
    <project-id>/  每个项目的上传文件
      video.mp4    原始视频
      audio.wav    提取的音频
      thumbnail.jpg 缩略图
      output.mp4   渲染输出
      transcript.json 转录原始数据
```
